# Optimizers

This is an application for showing how optimization algorithms work.

Это программа, написанная на Python с использованием библиотек **PySide6** для интерфейса,
**Matplotlib** для графиков и **NumPy** для расчётов.

Программа, можно сказать, является песочницей, в которой можно потестировать градиентные методы
нахождения минимума функции в трёхмерном пространстве.

## Возможности

- **29 оптимизаторов**: от классических SGD, Momentum, Nesterov и Newton до современных —
  Adam-семейство (AdamW, NAdam, RAdam, AdaBelief, Adan, ADOPT, AdEMAMix, CautiousAdamW,
  ScheduleFreeAdamW), Lion, Sophia, Prodigy, MARS, Shampoo, LBFGS, Левенберг—Марквардт и другие.
- **14 расписаний скорости обучения**: косинусный отжиг с тёплыми перезапусками, OneCycle,
  Noam, WSD, REX, циклическое, экспоненциальное и прочие; текущий lr виден в легенде анимации.
- **30 предустановленных тестовых функций** (Химмельблау, Розенброк, Растригин, Экли,
  Швефель, Eggholder…) плюс ввод произвольной функции двух переменных.
- Все глобальные минимумы функции находятся численно и отмечаются звёздами на графике.
- Графики в 2D и 3D (контуры или поверхность), анимация спуска со скоростью от ×0.125 до ×64.
- Настройка длины видимого хвоста траектории и скрытие/показ отдельных траекторий —
  удобно сравнивать оптимизаторы, идущие одним путём.
- Несколько оптимизаторов на одном графике, у каждого свои параметры и расписание.
- Интерфейс и график подстраиваются под светлую и тёмную тему системы —
  на скриншотах ниже обе.
- Новый оптимизатор можно добавить прямо из программы — достаточно класса с методом шага.

## Запуск

Требуется [uv](https://docs.astral.sh/uv/) — он сам установит Python 3.14 и зависимости:

```sh
uv run main.pyw
```

## Веб-версия

Та же песочница доступна и в браузере: FastAPI-бэкенд (`api/`) — тонкий шлюз
к неизменным `optimizers/`, `schedulers/` и `Function.py`, фронтенд —
React/TypeScript/Vite (`web/`).

По возможностям веб-версия повторяет десктопную — задание f(x, y), несколько
одновременных оптимизаторов со своими learning rate и расписанием, анимация
траектории, 2D-контуры и 3D-поверхность, выбор колормапа — и добавляет график
сходимости (значение функции от шага) под основным графиком.

Добавить новый оптимизатор или расписание в веб-версию можно так же, как и в
десктопной: положить `.py`-файл с классом-наследником в `optimizers/` или
`schedulers/` — бэкенд находит его тем же перебором атрибутов модуля
(`api/registries.py`), что и десктопный GUI. Никакого HTTP-эндпоинта для
загрузки или выполнения произвольного кода веб-бэкенд не предоставляет —
это осознанное ограничение ради безопасности, а не недоделанная функция:
через браузер добавить свой оптимизатор нельзя, только на сервере.

### Установка

```sh
uv sync
cd web && npm install
```

### Разработка

Нужны два процесса. Бэкенд:

```sh
uv run webapp.py
```

поднимает FastAPI на `127.0.0.1:58217`. Фронтенд — в отдельном терминале:

```sh
cd web && npm run dev
```

поднимает Vite dev-сервер на порту `47913`, который сам проксирует запросы
`/api` на бэкенд.

### Production

```sh
cd web && npm run build
```

собирает статику в `web/dist`; при следующем запуске `uv run webapp.py`
FastAPI сам обнаруживает эту папку и отдаёт фронтенд напрямую — отдельный
Vite-процесс уже не нужен.
